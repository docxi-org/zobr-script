# Покрытие

**Покрытие** суммирует весь вызов в одну цифру: из всего, что скрипт сделал, какая доля была [верифицирована](trust-classes) versus просто [утверждена](trust-classes)?

```
verified:  57%  ████████░░░
asserted:  43%  ██████░░░░░
```

## Как считается

Сервер считает события по [классам доверия](trust-classes) в конце каждого прогона:

- **verified** — события с доверием `verified` (sandbox-вызовы, контрольные точки, валидация формы conclude)
- **asserted** — события с доверием `asserted` (когнитивные операции, `report` с данными агента, `act`)
- **authority_gates** — события с доверием `authority` (ответы пользователя)
- **n/a** — инфраструктурные события (`start`, переходы статуса) — **исключаются** из подсчёта

Покрытие = `verified / (verified + asserted)`. Authority-события считаются отдельно, поскольку представляют внешние решения, а не рассуждение модели. Инфраструктурные события (`n/a`) исключаются полностью.

Ключевое различие: `report` считается **asserted**, потому что сервер лишь записывает то, что прислал агент — содержание произведено агентом. `checkpoint` считается **verified**, потому что сервер оценивает данные и отвечает директивой.

> **Совет:** покрытие — не оценка качества. Короткий полностью верифицированный прогон может быть тривиальным, а богатое asserted-исследование — именно тем, что нужно. Читайте покрытие вместе с [хронологией событий](trace), не отдельно.

## Где вы видите покрытие

- **Обзор** — полоса покрытия на каждой недавней трассировке
- **Список трассировок** — сортируемый столбец
- **Детали трассировки** — круговая диаграмма в панели итогов
- **Детали скрипта → Запуски** — покрытие по каждому вызову
