# Как работает запуск

**Вызов** (invocation) — это один прогон скрипта агентом. Вот что происходит:

## Цикл исполнения

1. **Регистрация** — агент вызывает `zs_register` со своим именем. Получает `agent_id` для всех последующих вызовов.
2. **Старт** — `zs_start` с указанием `script_ref`. Сервер загружает скрипт, создаёт вызов, возвращает когнитивный код + `invocation_id`.
3. **Исполнение** — агент читает код и выполняет каждую операцию (survey, doubt и т.д.), отправляя результаты через `zs_report`, `zs_checkpoint` и другие MCP-инструменты.
4. **Завершение** — `zs_conclude` с финальным результатом. Сервер валидирует его по объявленному типу и помечает вызов как `done`.

Каждый шаг записывается в [трассировку](trace). Агент передаёт `invocation_id` в каждом вызове, чтобы сервер знал, к какому прогону относится операция.

## Статусы

| Статус | Терминальный? | Как возникает |
|---|---|---|
| `running` | нет | Создан через `zs_start` — активное исполнение |
| `done` | да | `zs_conclude` прошёл успешно |
| `halted` | да | [Контрольная точка](checkpoints) вернула `halt` |
| `aborted` | да | Превышен бюджет, истёк TTL или вызван `zs_abort` |
| `errored` | да | Невосстановимый сбой |
| `expired` | да | Слишком долгое ожидание ввода (awaiting TTL) |
| `suspended` | нет | Приостановлен, ждёт внешний ответ (директива `ask`) |

Типичный поток: `running → done`. Контрольная точка с `halt` приводит к `halted`. Исчерпание бюджета — к `aborted`.

## Бюджеты и TTL

Каждый вызов работает в рамках лимитов:

- **Бюджет шагов** — макс. операций до принудительного прерывания (по умолчанию: 1000)
- **Бюджет итераций** — макс. итераций для циклов (по умолчанию: 100)
- **TTL вызова** — время жизни до вытеснения (по умолчанию: 1 час)
- **TTL ожидания** — как долго приостановленный вызов ждёт ввода (по умолчанию: 24 часа)

Превышение любого лимита завершает прогон. Подробнее в [Hot/cold жизненный цикл](hot-cold).

## Родительские и дочерние прогоны

Скрипт может порождать под-вызовы. Дочерние несут ссылку на родителя, а родитель перечисляет своих потомков. В детальном виде трассировки это дерево отображается для навигации по всему графу исполнения.
