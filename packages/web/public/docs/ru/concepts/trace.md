# Трассировка

Трассировка — **основной продукт** ZS-вызова. Не финальный ответ, а полная запись того, как этот ответ был получен. Каждый шаг агента, каждое решение сервера, каждая смена статуса записывается как событие.

## Структура события

```ts
{
  seq: 3,                          // порядковый номер
  t: "2026-06-01T19:29:05Z",      // временная метка ISO
  op: "report",                    // тип операции
  realizer: "server",             // кто произвёл
  trust: "verified",              // класс доверия
  inputs: [],                     // входные handle ID
  output: "h_abc123",            // выходной handle ID
  preview: "{ pattern, description, replaces }",
  meta: { label: "survey" }      // дополнительный контекст
}
```

У каждого события есть однострочный **превью**. Клик по нему в детальном виде раскрывает полный payload как подсвеченный JSON.

## Типы событий

| op | Когда | Доверие | Описание |
|---|---|---|---|
| `start` | Вызов создан | verified | Первое событие в каждой трассировке. |
| `report` | Агент отправляет телеметрию | verified | Промежуточное наблюдение — результат survey, doubt и т.д. |
| `checkpoint` | Серверный шлюз | verified | Возвращает [директиву](checkpoints): proceed, halt или ask. |
| `conclude` | Результат финализирован | verified | Результат валидирован по объявленной форме. |
| `status_transition` | Смена статуса | verified | Записывает `from → to` с причиной. |
| `ask_user` | Запрос ввода | authority | Скрипту нужен внешний ответ. |
| `act` | Внешнее действие | authority | Скрипт выполнил побочный эффект. |
| *(имя функции)* | Вызов серверной функции | verified | Метод [серверного модуля](server-module). |

## Чтение трассировки

Детальный вид показывает две панели:

- **Код** (слева) — когнитивный исходник с подсветкой строк. При наведении на событие подсвечивается соответствующая строка.
- **События** (справа) — хронология. Каждое событие показывает операцию, [бейдж доверия](trust-classes), превью и время. Раскрытие — полный JSON.

Под разделённым видом — **итоги покрытия**: круговая диаграмма [покрытия](coverage), счётчики событий и финальный результат.

## Что делает трассировку хорошей

Хорошо спроектированный скрипт создаёт трассировку, которая рассказывает историю: что было исследовано, что оспорено, где сервер верифицировал утверждения и как вывод следует из доказательств.
