# Агенты

**Агент** — любой MCP-клиент, выполняющий ZS-скрипты (обычно LLM в сессии чата). Перед любыми действиями агент должен зарегистрироваться.

## Регистрация

```
zs_register({ name: "Smith" }) → { agent_id: "ag_68de4eb2-813", role: "executor", guide: "...", active_invocations: [] }
```

Регистрация **идемпотентна по имени**: одно и то же имя всегда возвращает один `agent_id`. Агент может переподключаться между сессиями без потери идентичности. Ответ также содержит полный системный **guide** для роли агента — отдельный вызов `zs_guide()` не нужен.

`agent_id` обязателен на каждом последующем MCP-вызове (кроме `zs_register`). Сервер отклоняет вызовы с неизвестным `agent_id`.

## Чем владеет агент

- **Активные вызовы** — запущенные скрипты этого агента
- **История** — все прошлые вызовы (таблица invocations)
- **Блокировка записи** — во время активного вызова прямая запись в [хранилище](store) заблокирована

## Страница агентов

Страница **Агенты** в консоли показывает всех зарегистрированных агентов: имя, agent_id, дату регистрации, число активных вызовов, общее число запусков. Клик — переход к истории вызовов.

## Персистентность

Агенты хранятся в SQLite (таблица `zs_agents`). Переживают перезапуски сервера. Активные вызовы — эфемерные (в памяти), но могут быть восстановлены через [снимки](hot-cold).
