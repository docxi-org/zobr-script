# Библиотека

Библиотека — директория на сервере, хранящая все скрипты. Единственный источник правды о доступных скриптах.

## Файловая модель

Скрипт — это **файл**, не папка:

```
zs-lib/
  examples/
    hello.cog.ts                 ← script_ref = "examples/hello"
    insight.cog.ts               ← script_ref = "examples/insight"
    insight.srv.ts               ← парный серверный модуль
  analysis/
    market/
      trends.cog.ts              ← script_ref = "analysis/market/trends"
      trends.srv.ts
  zs.cognitive.d.ts              ← scaffold (автогенерация)
  zs.server.d.ts
  tsconfig*.json
```

- **`script_ref`** = путь от корня библиотеки до базового имени, без расширения
- Скрипт — это `.cog.ts` файл, опционально в паре с `.srv.ts` того же имени
- Папки — чистая группировка, любое количество скриптов на любой глубине вложенности

## Scaffold-файлы

При старте сервер записывает канонические файлы в корень библиотеки:

- `zs.cognitive.d.ts` — декларации типов когнитивных операций
- `zs.server.d.ts` — декларации типов базового класса ZsScript
- `tsconfig*.json` — конфигурации TypeScript для VS Code

Эти файлы перезаписываются при каждом рестарте. Исключение — `store.d.ts`: если он существует, сервер его не трогает (редактируется пользователем).

## Страница скриптов

Страница **Скрипты** в консоли показывает библиотеку в трёх видах:

- **Дерево** — сворачиваемая иерархия папок со счётчиками скриптов
- **Карточки** — визуальные карточки с описанием, числом запусков, последним запуском
- **Таблица** — сортируемый список с путём, бейджем сервера, статистикой

Описание на каждой карточке берётся из первого JSDoc-комментария (`/** ... */`) в `.cog.ts` файле.

## Создание скриптов

Новые скрипты создаются через страницу **Новый скрипт** или MCP-инструмент `zs_create`. Архитектор указывает папку (опционально) и имя. Сервер валидирует скрипт перед записью на диск.
